# memory/ README

This folder stores two kinds of memory:

1. **Daily memory** — `YYYY-MM-DD.md`
   - Raw session notes
   - Decisions
   - Useful context from ongoing work
   - Things explicitly worth remembering

2. **Technical memory** — durable files for recurring domain knowledge
   - Classification rules
   - Data conventions
   - Workflow defaults
   - Research context

## Recommended files

- `technical-profile.md` — high-level technical identity and tools
- `metallurgy-rules.md` — durable technical criteria and heuristics
- `workflows.md` — recurring analysis workflows
- `research-topics.md` — active themes, hypotheses, thesis directions

## Writing rule

If something is useful for only a day, put it in the daily note.
If it will matter again, promote it into a durable technical memory file or `MEMORY.md`.
