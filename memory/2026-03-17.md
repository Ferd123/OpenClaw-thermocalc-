# 2026-03-17

- Fernando quiere explorar un dashboard local para orquestar al asistente con dos trabajos en serie.
- Idea base acordada: arquitectura en 3 capas:
  - dashboard (UI para crear/ver tareas y revisar estados/logs)
  - orquestador (coordina el pipeline secuencial)
  - workers (ejecutan Codex y Gemini)
- V1 propuesta:
  - frontend simple
  - backend con API
  - jobs de 2 pasos en serie
  - selección de agente por paso (Codex/Gemini)
  - estados: pendiente, corriendo, terminado, error
  - logs y resultado final
  - aprobación manual opcional antes del paso 2
  - persistencia en SQLite
- Stack sugerido:
  - backend: Python + FastAPI
  - frontend: React + Vite
  - persistencia: SQLite
- Casos de uso conversados:
  - Gemini planifica / Codex implementa
  - Codex escribe código / Gemini revisa, resume o documenta
